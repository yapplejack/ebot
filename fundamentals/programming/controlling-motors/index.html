Controlling Motors

The goal of this section is to allow you to control and verify the operation of your motors. Motors are controlled by motor controllers which require unique CAN IDs that are referenced in the code. When setting up our motor controllers we have the ability to assign these CAN IDs, ensure that the controller is setup to work with the connected motor and power the motor if appropriate.


Later on you will want to test motors with code, to do this we will need to have installed the current version of WPILib, FRC Game Tools, the proper software for the motor controller and have updated the roboRIO with the newest imaged.


note

You may test your SPARK MAX controllers on compatible motors with controller's USB port and robot power without these installations.


When setting up motor controllers is generally good practice to have a test bench or the electronics layed out on your uninstalled bellypan. See WPILib's Introduction to FRC Robot Wiring for instructions on setting a test bench/ bellypan like the one shown below. If you are putting together an Everybot our Bill of Materials will have everything needed to create the Everybot's bellypan but may not have everything required to build their setup.


Setting up a Spark MAX​

TODO: improve pictures for sparkmax section

To work with the Spark MAX motor controllers please download the REV Hardware Client if you have not done so already. We recommend using the official documentation from REV at the linked website but we will go over some key points below.



Interfacing with the Spark MAX is done through the USB-C slot, there is an included cable but its short length makes it annoying to use, so a better cable is a good purchase.



When working with Spark MAXes in CAN we recommend to make sure all Spark MAXes have the newest firmware individually without the presence of other controllers in the CAN loop. This can be done simply by not powering on the robot and connecting to each controller individually via USB. When connected, open the REV Hardware Client and select Spark MAX. Then navigate to the update tab and select download if there is a red Out-Of-Date triangle. This step can be done at any time but additional steps will need to be taken once you have figured out what motor your SPARK MAX will be connected to and where it will be on the robot.



There are many important settings that will impact the performance of your motor but we want to highlight the two most important settings: It is imperative that CAN ID is set correctly. When Spark MAXes are declared in code they require a CAN ID to be initialized with, make sure to change the CAN ID in the client to match the desired motor in your code. Consider labeling the motor controllers with a sticker indicating their CAN ID. If you are unsure which motor controller you are modifying you can make the LED on the SPARK MAX blink via the button to the left of the CAN ID. The CAN ID can be number from 1 to 63 and must be unique.


The other important setting is Motor Type, if you are NOT using a NEO motor this must be set to Brushed Motor, otherwise if using a NEO it should be REV NEO Brushless. When finished configuring the motor controller make sure to click the Burn Flash button.



If you would like to test your motor without the presence of code read through REV's Make it Spin!. Pay special attention to the limiting current section and make sure to take proper precautions before running your motors. These precautions may include:


  • Chocking the robot up with something sturdy such that no wheels can touch the floor if testing the drive train
  • Making sure that hardstops are in place if a mechanism can collide with the robot, floor or over-rotate
  • Ensuring nobody is near the robot and alerting everyone the robot is about to be run
  • Verifying that the motor is wired correctly and the motor controller is setup for the motor
  • Preventing the motor from moving if not connected to a mechanism

When a motor does not run or operate as expected it may be prudent to check the Status LED of the SPARK MAX. If the status LED is expected check out REV's troubleshooting section or reach out for help.

Setting up a Victor SPX​

To work with the CTRE motor controllers please download the Phoenix Installer in the What to Download section if you have not done so already. We recommend using the official documentation from CTRE at the linked website but we will go over some key points below.


CTRE highly recommends that you test your hardware when you receive it. If you are close to completing your robot and just arrived at this step it can be skipped with one key exception: You are using your own code to test the robot or are not using the provided Everybot code.


To create a test program for your hardware please see FRC: VS CODE C++/Java and for those of you writing your own code please see the FRC C++/Java - Add Phoenix section. Make sure that your declared motor controller reflects the one you have, if you are using VictorSPXs and following along with CTRE's documentation you can replace their TalonSRXs with the following:


/src/main/java/frc/robot/Robot.java

// when importing
import com.ctre.phoenix.motorcontrol.ControlMode;
import com.ctre.phoenix.motorcontrol.can.VictorSPX;

//In the class timed robot
VictorSPX myvictor = new VictorSPX(0);
// class name | variable name | = | new keyword | class constructor with desired CAN ID in parenthesis | ;

// typically the variable name should be more descriptive: m_frontRightDrive or just m_frontRight for a drive motor, m_arm for the motor that controls an arm, etc.

//In robotInit()
myvictor.set(ControlMode.PercentOutput, 0);

Please note that this will not spin the motor as we have set the output power to zero, this value can be a value from -1 to 1 (.5 = 50% power in one direction and -.5% = -50% power in the opposite direction).


Now we will need to Bring Up the Can Bus. CTRE first recommends that you check your wiring, then powering on the robot to verify that all motor controllers have lit LEDs. Launch Phoenix Tuner and navigate to the CAN Devices tab:



You should see the motor controller LEDs light up, chance are they will be red and need to have a unique CAN ID set. CTRE recommends you physically connect motor controllers to roboRIO via the CAN wire one by one, making that robot is powered off when rewiring.


From here we will need to set the CAN ID of the motor controller. When CTRE motor controllers are declared in code they require a CAN ID to be initialized with, make sure to change the CAN ID in the client to match the desired motor in your code by setting the value and hitting the change ID button, which should now be blue. After completion there should be an ok in the bottom right. Consider labeling the motor controllers with a sticker indicating their CAN ID.



After assigning all CTRE motor controllers a unique CAN ID you will need to update their firmware. Please refer to CTRE's guide for this.
